### Strategy Description

#### I. Summary
Data is the source of quantitative trading, and efficiently managing a large volume of data is a critical aspect. Databases are one of the best solutions for this challenge, with applications now being standard configurations in intraday and high-frequency trading strategies. In this paper, we will explore the built-in database system by Inventor Quant (FMZ.COM), covering how to create tables, store data, modify data, delete data, reference data, and apply it to practical scenarios.

#### II. How to Choose a Database
Those familiar with the Inventor Quant platform know that before, saving data locally for reuse could only be done using the `_G()` function. Each time the strategy is stopped, this function would automatically save necessary information. However, if more complex and formatted data needed to be saved, the `_G()` function was not sufficient. Hence, many turned to self-built databases as a solution.

Talking about self-built databases, one can think of Oracle, MySQL, KDB, OneTick, NoSQL... These are all excellent enterprise-level applications with powerful functions and performance. However, they come with challenges: steep learning curves, complex configurations, and maintenance difficulties. For individual traders in quantitative trading, these might be overkill—though once learned, their functionality is limited.

#### III. Inventor Quant Built-in Database
Next, let's get to know the lightweight database built into Inventor Quant, which is DBExec—a relational database management system interface developed based on SQLite. It is written in C and has a small footprint with low resource usage, making it perfect for local data management by financial quant analysis enthusiasts. Different "objects" (such as exchanges, data sources, prices) can be divided into different tables, relationships defined between them. Moreover, no separate installation or configuration is required; just call the DBExec() function to use!

Additionally, learning SQLite language has a low cost. Most of the work on databases is done by SQLite statements. Familiarity with basic syntax suffices for most needs. Here are some fundamental SQLite commands.

#### IV. Fundamental Syntax
SQLite syntax is case-insensitive, although certain commands are sensitive (e.g., `GLOB` and `glob` have different meanings). SQLite statements can start with any keyword such as `SELECT`, `INSERT`, `UPDATE`, `DELETE`, `ALTER`, `DROP`, which represent: fetching data, inserting data, updating data, deleting data, modifying the database, and deleting tables, respectively. All statements end with a semicolon. Here is an example of creating, adding, removing, updating, and querying a database:
```cpp
function main() {
    // Create table if it doesn't exist: "users" with columns id (integer primary key auto-increment), name (text not null)
    Log(DBExec('CREATE TABLE IF NOT EXISTS "users" (id INTEGER PRIMARY KEY AUTOINCREMENT, name text not NULL);'));
    
    // Add data:
    Log(DBExec("INSERT INTO users(name) values('Zhang San')"));
    Log(DBExec("INSERT INTO users(name) values('Li Si')"));
    
    // Delete data:
    Log(DBExec("DELETE FROM users WHERE id=1;"));
    
    // Update data:
    Log(DBExec("UPDATE users SET name='Wang Wu' WHERE id=2"));
    
    // Query data:
    Log(DBExec('select 2, ?, ?, ?, ?', 'ok', true,9.8,null));
    Log(DBExec('select * from kvdb'));
    Log(DBExec('select * from cfg'));
    Log(DBExec('select * from log'));
    Log(DBExec('select * from profit'));
    Log(DBExec('select * from chart'));
    Log(DBExec("selEct * from users"));
}
```
A database typically contains one or more tables, each identified by a name. Notable system reserved names are: `kvdb`, `cfg`, `log`, `profit`, and `chart`. Hence, when creating tables, avoid using these reserved names. Running the code above will yield:

#### V. Strategy Example
After understanding the fundamental SQLite syntax, let's use Inventor Quant’s built-in database to create a strategy for collecting and using tick data.

**Step 1: Update Host**
Ensure you are using the latest version of the host; if you have previously downloaded and used it, first delete it and download and deploy again from the https://www.fmz.cn/m/add-node page.

**Step 2: Create Strategy**
```cpp
function main() {
    // Subscribe contract
    _C(exchange.SetContractType, 'swap');
    
    // Create data table
    DBExec('CREATE TABLE IF NOT EXISTS "tick" (id INTEGER PRIMARY KEY AUTOINCREMENT,'.concat(
        'High FLOAT not NULL,', 
        'Low FLOAT not NULL,', 
        'Sell FLOAT not NULL,', 
        'Buy FLOAT not NULL,', 
        'Last FLOAT not NULL,', 
        'Volume INTEGER not NULL,', 
        'Time INTEGER not NULL);'
    ));
    
    // Get 10 tick data points
    while (true) {
        let tick = exchange.GetTicker();
        // Add data to the "tick" table
        DBExec(`INSERT INTO tick(High, Low, Sell, Buy, Last, Volume, Time) values(${tick.High}, ${tick.Low}, ${tick.Sell}, ${tick.Buy}, ${tick.Last}, ${tick.Volume}, ${tick.Time})`);
        // Query all data
        let allDate = DBExec('select * from tick');
        if (allDate.values.length > 10) {
            break;
        }
        Sleep(1000);
    }
    
    // Query all data
    Log(DBExec('select * from tick'));
    
    // Query the first piece of data
    Log(DBExec('select * from tick limit 1'));
    
    // Query the first two pieces of data
    Log(DBExec('select * from tick limit 0,2'));
    
    // Delete the first piece of data
    Log(DBExec('DELETE FROM tick WHERE id=1;'));
    
    // Modify the second piece of data
    Log(DBExec('UPDATE tick SET High=10000 WHERE id=2'));
    
    // Query all data again
    let allDate = DBExec('select * from tick');
    Log(allDate);
}
```

**Step 3: Run Strategy**
For example, on Windows, after running the strategy, a folder named by the robot number will be generated in the host directory's "logs\storage" path. Open this folder to find a file with the ".db3" extension; that is the Inventor Quant built-in database file as shown:
![IMG](https://www.fmz.com/upload/asset/39b811e1f9df0a2911f9.png)

The code first creates a "tick" table, adds tick data fields to it, and then populates the "tick" table with fetched tick data. The loop stops when more than 10 pieces of data are stored. Finally, the script performs operations using five SQLite commands—querying, deleting, modifying data in the database—and logs the results as shown:
![IMG](https://www.fmz.com/upload/asset/395a055a194ff230e4a7.png)

    // Create status bar
